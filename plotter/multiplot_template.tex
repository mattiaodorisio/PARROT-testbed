The following figures shows the performance of DeLI index under different configurations. The benchmark has been executued as follows: for each dataset, we run the three workloads (in-distribution lookup, out-of-distribution lookup, and in-distribution insertion) on a batch of $k = 2^14$ operations (there is no particular reason for this value, we can discuss a better choice). The plots below show the sum to execute the batch of $k$ operations.
The workloads are:
\begin{itemize}
  \item In-distribution Lookup: searching for $k$ keys already present within the dataset.
  \item Out-of-distribution Lookup: searching for $k$ keys that do not belong to the same dataset, but whose distribution follows the same of the dataset.
  \item In-distribution Insertion: inserting $k$ keys drawn from the same distribution of the dataset.
\end{itemize}

The following plots compare the very many possible configuartions of the DeLI index. Each configuration is defined by the following parameters:
\begin{itemize}
  \item dynamic: all the plots refer to the dynamic case.
  \item bottom level optimization
  \item simd lanes (0 for no simd)
  \item rht load factor
  \item top level optimization
  \item number of high bits: we discussed to take all the bits in [1, 20]. Due to the huge time taken to run them all, for the moment i just took 4, 8, 12, 16, and 20.
\end{itemize}

\section{Comparisons among indexes}
The plots below compare the performance of different indexes on various datasets by varying the initial dataset size. Please note, that the dataset size can change due to updates. 
All the workloads are run as a batch of $k$ operations. Each batch is repeated multiple times to obtain median of the execution time.
The workloads are:
\begin{itemize}
  \item In-distribution Lookup: searching for $k$ keys already present within the dataset.
  \item Out-of-distribution Lookup: searching for $k$ keys that do not belong to the same dataset, but whose distribution follows the same of the dataset.
  \item In-distribution Insertion: inserting $k$ keys drawn from the same distribution of the dataset.
\end{itemize}
